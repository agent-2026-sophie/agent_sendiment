> 两者读音不同：**pass@k** 读作 "pass at k"，**pass^k** 读作 "pass wedge k"（楔形/幂）。它们看起来只差一个符号，但衡量的是**两件几乎相反的事**——一个衡量"能力上限"，一个衡量"可靠性下限"。

---

## 1. 一句话区分

| 指标 | 核心问题 | 通俗解释 |
|------|----------|----------|
| **pass@k** | 跑 k 次，**至少成功一次**的概率 | "只要让它多试几次，它**能不能**做出来？" —— 衡量**能力 / 上限** |
| **pass^k** | 跑 k 次，**每一次都成功**的概率 | "同一个任务连续跑 k 次，它**能不能次次都对**？" —— 衡量**一致性 / 可靠性** |

随着 k 增大，**pass@k 单调上升**（机会越多越容易撞对），而 **pass^k 单调下降**（要求越多越难全对）。两条曲线的走向恰好相反，这正是两者的本质差异[[The Reliability Gap: Agent Benchmarks for Enterprise]](https://simmering.dev/blog/agent-benchmarks)。

## 2. pass@k：含义与计算

### 2.1 定义

pass@k 是 OpenAI 在 Codex / HumanEval 论文（Chen et al., 2021）中提出的代码生成评估指标，定义为：**针对一个问题独立采样 k 个候选答案，其中至少有一个通过全部单元测试（功能正确）的概率**[[HumanEval: LLM Benchmark for Code Generation]](https://deepgram.com/learn/humaneval-llm-benchmark)。

注意常见误解：**pass@1 不是"第一次就答对"，而是"单次采样答对的概率"**；pass@k 也不是"第 k 次才通过"[[Statistics for AI/ML, Part 4: pass@k and Unbiased Estimator]](https://leehanchung.github.io/blogs/2025/09/08/pass-at-k)。

### 2.2 朴素估计（有偏）

最直接的想法是用经验成功率 $\hat{p}$（正确样本数 ÷ 总样本数）代入：

$$\text{pass@}k = 1 - (1-\hat{p})^k$$

但这是**有偏估计**：$(1-\hat{p})^k$ 假设每次抽样相互独立（有放回 / 无限总体）。而实际是从 n 个已生成样本里**无放回**地取 k 个，抽样不独立，会**系统性低估**真实 pass@k[[Statistics for AI/ML, Part 4: pass@k and Unbiased Estimator]](https://leehanchung.github.io/blogs/2025/09/08/pass-at-k)。

### 2.3 无偏估计（OpenAI 标准做法）

为消除偏差，OpenAI 采用基于 **U 统计量**（U = Unbiased）的无偏估计：生成 n 个样本，其中 c 个正确，则

$$\text{pass@}k = 1 - \frac{\dbinom{n-c}{k}}{\dbinom{n}{k}}$$

- $n$：总样本数
- $c$：正确样本数
- $k$：考察的采样数（$k \le n$）

含义：$\binom{n-c}{k}/\binom{n}{k}$ 是"抽出的 k 个**全部来自 c 个错误样本**"的概率，1 减去它就是"至少一个正确"的概率。

**特例 k=1**：$\text{pass@}1 = 1 - \frac{n-c}{n} = \frac{c}{n}$，即经验成功率本身（此时有放回与无放回无差异）。

### 2.4 数值示例

设 $n=10$，$c=3$（30% 正确率），求 pass@5：

- 朴素（有偏）：$1 - 0.7^5 = 0.8319$
- 无偏：$1 - \dfrac{\binom{7}{5}}{\binom{10}{5}} = 1 - \dfrac{21}{252} = 0.9167$

无偏估计（91.7%）明显高于朴素估计（83.2%）；**样本量越小、k 越接近 n，二者差距越大**[[Statistics for AI/ML, Part 4: pass@k and Unbiased Estimator]](https://leehanchung.github.io/blogs/2025/09/08/pass-at-k)。

## 3. pass^k：含义与计算

### 3.1 定义

pass^k 表示：**同一个任务独立运行 k 次，k 次全部成功的概率**。它在实践中通过经验法测量——把每个任务跑 k 遍，统计"k 遍全对"的任务占比；曲线下降越陡，说明跨多次运行的表现越不稳定[[The Reliability Gap: Agent Benchmarks for Enterprise]](https://simmering.dev/blog/agent-benchmarks)。

### 3.2 简化公式

若单次成功概率为 $p$ 且各次独立，则

$$\text{pass}^k = p^k$$

例如单次成功率 80%（pass^1 = 0.8）：
- pass^2 = 0.64
- pass^4 ≈ 0.41
- pass^8 ≈ 0.168

可见即便单次表现不错，**连续多次全对的概率会迅速衰减**——这正是 pass^k 想暴露的"可靠性问题"。

### 3.3 典型用途

pass^k 主要出现在 **Agent / 多轮交互类基准**中，用来量化可靠性。例如 **τ²-bench**（客服 Agent 基准）系统性报告 pass^k：顶尖模型 pass^1 可达 ~80–85%，但到 pass^4 / pass^8 会显著下滑，意味着"今天处理得完美，明天可能就失败"[[The Reliability Gap: Agent Benchmarks for Enterprise]](https://simmering.dev/blog/agent-benchmarks)。

## 4. 核心差别对比

| 维度 | pass@k | pass^k |
|------|--------|--------|
| 成功判据 | k 次中**至少一次**成功 | k 次**全部**成功 |
| 随 k 变化 | **单调递增**（趋近上限） | **单调递减**（趋近 0） |
| 衡量本质 | 能力上限 / 可达性（capability） | 一致性 / 可靠性（reliability） |
| 简化公式（独立假设） | $1-(1-p)^k$ | $p^k$ |
| 标准无偏公式 | $1-\binom{n-c}{k}/\binom{n}{k}$ | 一般用经验频率直接测 |
| 典型场景 | 代码/数学生成（HumanEval、MBPP、AIME），可多次采样 + 验证器筛选 | Agent / 客服 / 长程任务（τ²-bench 等）一致性评估 |
| 隐含使用方式 | 允许"采样多个、择优"（Best-of-N、自洽投票） | 要求"每次都得对"，不能靠重试兜底 |
| 关注的问题 | "它**能**做到吗？" | "它**每次**都能做到吗？" |

---

## 5. 怎么选 / 怎么用

- **看模型潜力、做研究对比**：用 **pass@k**。pass@1 反映单次实力，pass@k（k=10/100）反映"多采样 + 验证器筛选"的上限。pass@10 远高于 pass@1，说明该任务适合用**多采样择优 / 自洽投票 / Best-of-N**等推理策略提升可靠性（代价是延迟和成本）[[Statistics for AI/ML, Part 4: pass@k and Unbiased Estimator]](https://leehanchung.github.io/blogs/2025/09/08/pass-at-k)。
- **看生产落地、做业务自动化**：用 **pass^k** 更相关。企业自动化关心的不是"试很多次能不能成"，而是"放它独立干、每次都靠谱吗"。遗憾的是大多数基准只报告 pass^1，少有报告更高阶 pass^k[[The Reliability Gap: Agent Benchmarks for Enterprise]](https://simmering.dev/blog/agent-benchmarks)。
- **可重试 vs 不可重试**：能廉价重试并自动验证结果的场景（如代码生成 + 单测），pass@k 更贴合实际收益；无法重试、错一次即代价高昂的场景（客服、交易、自动下单），pass^k 才是关键约束。

---

## 6. 一图理解

假设某模型在某任务上单次成功率 p = 0.8：

| k | pass@k（≥1 次成功） | pass^k（k 次全对） |
|---|---------------------|--------------------|
| 1 | 0.80 | 0.80 |
| 2 | 0.96 | 0.64 |
| 4 | 0.9984 | 0.41 |
| 8 | ≈1.000 | 0.168 |

同一个模型、同一个 p，**pass@k 越跑越接近 1，pass^k 越跑越接近 0**——这张表把两者的本质对立关系展现得最清楚。
